# EPIC_GAME.md — Party Word Game / Alias-like Android App

## 1. Цель

Создать Android-приложение для командной словесной игры, где игроки объясняют, показывают или рисуют слова из заранее сформированной пачки.

Ключевые требования:

- быстрый старт игры;
- минимальный UX (передача телефона);
- оффлайн-режим;
- честный баланс между командами.

---

## 2. Core Idea

В приложении есть общий словарь слов.

Перед началом игры пользователь задаёт размер пачки (bulk):

- 40 / 60 / 80 / 100 слов

Приложение случайно выбирает слова из словаря.

Эта пачка используется на протяжenne всей игры и всех раундов.

---

## 3. Sessions и уникальность слов

Если существует несколько игровых сессий:

- слова не должны пересекаться между активными пачками.

### Поведение при нехватке слов

Если слов недостаточно:

- показать предупреждение;
- предложить уменьшить размер пачки;
- либо разрешить пересечение через настройку.

---

## 4. Настройки игры

Перед стартом пользователь задаёт:

- размер пачки слов;
- количество команд;
- количество раундов;
- длительность хода (по умолчанию 60 секунд).

### Упрощение MVP

- количество участников НЕ учитывается;
- имена игроков НЕ требуются;
- телефон передаётся вручную.

---

## 5. Основной игровой цикл

Игра делится на раунды.

Внутри раунда команды ходят по очереди.

Пример:

1. Команда 1 — 60 секунд
2. Команда 2 — 60 секунд
3. Команда 1
4. Команда 2

И так далее.

Раунд продолжается до тех пор, пока не будут угаданы ВСЕ слова из пачки.

---

## 6. Жизненный цикл слова

Каждое слово в раунде имеет состояние:

```text
available   — доступно для игры
in_turn     — сейчас показывается
guessed     — угадано (исключается до следующего раунда)
````

Важно:

* слово не удаляется из раунда, пока его не угадают;
* слово может переходить между командами.

---

## 7. Ход игрока

### До старта

Экран:

* команда
* номер раунда
* правило раунда
* кнопка `Старт`

### После старта

Отображается:

* текущее слово
* таймер
* кнопки:

  * `Угадали`
  * `Не угадали`
  * `Следующее слово`
  * `Пауза`

---

## 8. Обработка действий

### Угадали

* команда получает +1 очко;
* слово → `guessed`;
* слово исключается до следующего раунда;
* показывается новое слово.

---

### Не угадали

* очки не начисляются;
* слово остаётся в раунде;
* слово может быть угадано другой командой.

---

### Следующее слово (пропуск)

Работает так же как `Не угадали`:

* без очков;
* слово остаётся в игре;
* переходит другим командам.

---

## 9. Передача слова между командами

Если команда не угадала слово:

* оно остаётся в пуле;
* следующая команда может получить это же слово.

Слово «живёт», пока не будет угадано.

---

## 10. Конец хода

Ход заканчивается:

* по таймеру;
* или вручную.

После этого:

* фиксируются очки;
* ход передаётся следующей команде.

---

## 11. Конец раунда

Раунд заканчивается, когда:

* ВСЕ слова из пачки получили статус `guessed`.

После этого:

* показывается summary;
* начинается новый раунд;
* все слова снова становятся `available`.

---

## 12. Раунды

### Round 1 — Объяснение

Нельзя:

* говорить слово;
* использовать однокоренные.

---

### Round 2 — Крокодил

Только жесты.

---

### Round 3 — Одно слово

Можно сказать только одну подсказку.

---

### Round 4 — Рисование (бонус)

Для MVP:

* без встроенного рисования;
* игрок рисует оффлайн.

## 20. Финальные решения MVP

* команды есть, игроки не учитываются;
* одна пачка на всю игру;
* пачки не пересекаются между сессиями;
* слова живут до угадывания;
* пропуск = не угадали;
* слово даёт очко только один раз за раунд;
* раунд = пока не угадали всё;
* затем всё повторяется в следующем раунде.

## 21. Implementation Plan

### Data Models

#### Word
Represents a single word in the game with its state.

#### Team
Represents a team with their score.

#### GameSettings
Configuration settings for a game session.

#### GameState
Current state of the game including teams, words, and current turn.

### Core Logic

#### GameManager
Main class responsible for game logic:
- Starting a new game
- Managing turns
- Handling word states
- Calculating scores
- Moving between rounds

#### WordDictionary
Manages the collection of available words and selection of word batches.

#### GamePersistence
Handles saving/loading game state to/from storage.

### UI Components

#### HomeScreen
Start a new game or continue existing one.

#### SetupScreen
Configure game settings.

#### GameScreen
Main gameplay screen showing current word and timer.

#### SummaryScreen
Display round/team results.

#### FinalScreen
Show final results and winner.

4. Create persistence layer
